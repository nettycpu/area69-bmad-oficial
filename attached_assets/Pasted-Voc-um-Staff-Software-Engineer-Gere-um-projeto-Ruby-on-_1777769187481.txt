PROVIDERS
1) Wavespeed AI (webhook maduro)
- Ao submeter job, enviar "webhook" na requisição (URL do nosso backend).
- Webhook de retorno: POST no nosso endpoint com assinatura HMAC-SHA256.
Headers:
  - webhook-id
  - webhook-timestamp (unix)
  - webhook-signature: "v3,<hmac_hex>"
Regras:
  - Verificar replay attack: timestamp dentro de 300s.
  - Deduplicar por webhook-id (unique por provider + webhook_id).
Payload: status, outputs (array de URLs), executionTime, error.
Documentação (somente referência; não copie HTML):
- https://wavespeed.ai/docs/how-to-use-webhooks
- https://wavespeed.ai/docs/verify-webhooks

2) Higgsfield Cloud (webhook existe, menos previsível)
- Ao submeter job: enviar webhook_url no payload.
- Também existe polling: GET /v1/generations/{id} (usar como fallback/safety net).
- Como assinatura não é confirmada, proteger webhook com token secret:
  /v1/webhooks/higgsfield?token=...
E também aplicar rate limiting no endpoint e logs.

ENDPOINTS (rotas versionadas)
Base: /v1

Auth:
- Todas rotas /v1/* (exceto webhooks e health) exigem Authorization: Bearer <clerk_jwt>.
- Validar JWT via JWKS (CLERK_JWKS_URL). Extrair sub como user_id.
- Rejeitar se inválido.

Health:
- GET /health -> 200 {"ok":true}

Generations:
- POST /v1/generations
  - header: Idempotency-Key (obrigatório)
  - body: { provider, kind, prompt?, params? }
  - cria Generation com status="pending", expires_at=now+7days
  - se existir Generation com mesmo (user_id, idempotency_key), retornar o existente (idempotente)
  - enfileirar job SubmitGenerationJob(generation_id)
  - retorna 202 { id, status }

- GET /v1/generations/:id
  - só permite acessar se user_id dono
  - retorna { id, status, result_urls?, error? , expires_at }

- GET /v1/generations
  - lista do user_id, ordenado desc
  - paginação simples com limit + cursor (created_at + id) para não virar dívida futura

Avatar trainings:
- POST /v1/avatar_trainings
  - header: Idempotency-Key (obrigatório)
  - body: { photos_count, input_photos_refs: [...] , params? }
  - cria AvatarTraining status=pending
  - enfileira SubmitAvatarTrainingJob(training_id)
  - retorna 202

- GET /v1/avatar_trainings/:id (owner only)
- GET /v1/avatar_trainings (owner only, paginado)

Webhooks (sem auth Clerk):
- POST /v1/webhooks/wavespeed
  - verificar assinatura HMAC-SHA256 v3 com WAVESPEED_WEBHOOK_SECRET
  - replay: timestamp dentro 300s
  - dedup por webhook-id (salvar WebhookEvent)
  - parse JSON
  - localizar Generation por provider="wavespeed" + provider_job_id (do payload)
  - atualizar status/result_urls/execution_time_ms/error
  - responder 200 rápido (sem bloquear)
  - qualquer erro de segurança -> 401

- POST /v1/webhooks/higgsfield?token=...
  - validar token == HIGGSFIELD_WEBHOOK_TOKEN
  - dedup por payload_hash = SHA256(raw_body)
  - parse JSON e tentar extrair provider_job_id
  - atualizar Generation/AvatarTraining correspondentes
  - responder 200 rápido

JOBS (Solid Queue)
- SubmitGenerationJob
  - lê Generation
  - marca status=submitted, started_at=now
  - chama client do provider (Wavespeed/Higgsfield) e obtém provider_job_id
  - salva provider_job_id
  - (Wavespeed) enviar webhook URL: WAVESPEED_WEBHOOK_BASE_URL + "/v1/webhooks/wavespeed"
  - (Higgsfield) enviar webhook_url: HIGGSFIELD_WEBHOOK_BASE_URL + "/v1/webhooks/higgsfield?token=..."
  - lidar com falhas: status=failed com error_message

- SubmitAvatarTrainingJob
  - similar: submete treino, salva provider_job_id, status=submitted/running

- WatchdogPollHiggsfieldJob
  - roda periodicamente (documentar no README como agendar)
  - pega jobs Higgsfield em submitted/running sem update há X min
  - chama GET status no Higgsfield e atualiza
  - usar backoff e limitar quantidade por execução (ex: 50 por run)

- ExpireGenerationsJob
  - roda periodicamente
  - pega generations completed com expires_at <= now e status != expired
  - limpa result_urls, marca status=expired, expired_at=now
